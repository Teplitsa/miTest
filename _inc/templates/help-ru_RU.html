Создание тестов с использованием плагина FRL MITEST

Плагин позволяет создавать интерактивный тест для оценки качества веб-сайтов. Каждый тест трактуется как модуль, состоящий из серии вопросов. По завершении тестирования, пользователь может запросить пересылку результатов по email, оставить свой комментарий и запросить консультацию по теме тестирования.

Создание модулей

Сначала необходимо создать такой модуль в разделе MITEST -> Модули. Интерфейс создания модуля аналогичен интерфейсу управления любой таксономией в WordPress. Для описания модуля предусмотрены следующие поля

  • Название - заголовок модуля
  • Ярлык - наименование, используемое в URL
  • Описание - кратное описание модуля, используемое в списке модулей
  • Правила - подробное описание и правила прохождения теста, отображаются перед началом тестирования
  • Шкала результатов - данное поле позволяет задать шкалу, по которой производится оценка полученного в ходе теста результата, и описание интервалов этой шкалы.

Для определения каждого интервала внутри шкалы результатов используются спецкоды двух типов:

1.- Для описания цветового блока (шкала в целом)

[res_scale grades="12,20,40,80"]

Значения параметра grades представляют собой пороговые значения интервалов (стартовый "0" можно не указывать).

2.- Для описания каждого отдельного интервала

[m_result bottom="0" top="12"]Текст описания. [/m_result]

Значения параметров bottom и top представляют собой значения нижней и верхней границ интервала соответственно. Текст, указываемый между тегами спецкода, представляет собой комментарий (описание) интервала.

Для вставки спецкодов в поле редактирования можно использовать выпадающее меню Коды тестов.

Если к модулю уже присоединены вопросы, можно просмотреть их тексты и варианты ответов с помощью опции Просмотр, расположенной в контекстном меню под заголовком модуля. С помощью опции Перейти можно осуществить переход к началу тестирования по выбранному модулю.

Создание вопросов

После создания модуля, необходимо внести составляющие его вопросы и возможные опции ответов, используя разделы MITEST -> Вопросы и/или MITEST -> Добавить вопрос. Интерфейс создания и редактирования вопроса в целом аналогичен интерфейсу создания и редактирования записей в WordPress. Он предусматривает заполнение следующих полей:

  • Текст вопроса - собственно формулировка вопроса
  • Основное содержание - поле, предназначенное для ввода опций ответа
  • Дополнительные данные: Тип вопроса - позволяет выбрать тип вопроса: мультивариантный (допускает возможность выбора нескольких вариантов ответа), единственный вариант (допускает выбор только одного варианта ответа)
  • Дополнительные данные: Сортировка - числовой индекс для сортировки вопросов внутри модуля (вопросы сортируются по возрастанию этого показателя)
  • Дополнительные данные: Комментарий - текст пояснительного комментария к вопросу
  • Дополнительные данные: Ссылки - список ссылок по теме вопроса

Поля "Комментарии" и "Ссылки" предполагают использование стандартных HTML-тэгов для форматирования содержания (включая список ссылок). Основное содержание предполагает использование спецкода для описания опций ответов:

[q_opt id="6" score="4" cfield="yes"]Ничего из вышеперечисленного[/q_opt]

Спецкод принимает следующие параметры: - id - порядковый номер опции внутри вопроса (положительно число, больше нуля) - score - количество баллов, начисляемое за выбор опции (положительно число, больше нуля) - cfield - наличие поля для пользовательского комментария (допустимые значения: yes/no, по умолчанию принимаеся no)

Текст, указываемый между тегами спецкода, представляет собой собственно текст опции для выбора в качестве ответа на вопрос.

Для вставки спецкодов в поле редактирования можно использовать выпадающее меню Коды тестов.

В блоке "Модули" необходимо указать, к какому модулю относится вопрос. Начните писать название модуля в поле ввода и затем выберите подходящий их предложенных вариантов. Если такого варианта нет, можно создать новый модуль, указав его название полностью и нажав "Добавить". После установки модуля сохраните изменения.

Настройки плагина

Раздел MITEST -> Настройки, позволяет указать следующие основные настройки плагина

  • "От" email - email для использования в поле "от" для сообщений, отправляемых плагином (по умолчанию используется соответствующая настройка WordPress)
  • Email для извещений - email, на который будут отправляться уведомления о прохождении теста (по умолчанию используется адрес администратора сайта из настроек WordPress)
  • Письмо участнику - текст письма, отправляемого пользователю, прошедшему тест и выбравшему опцию пересылки результатов. В тексте необходимо использовать спецкод [link], который при формировании письма будет заменен ссылкой на профиль ответов.
  • Информация о разработчиках - текст, выводимый в нижней части шаблонов теста и просмотра результатов с информацией об авторах и правах использования.
  • Общие правила - текст правил, общих для всех вопросов и прохождения теста в целом, выводится для пользователя перед началом тестирования.
  • Заставка страницы теста - позволяет разместить специальное изображение для заголовков страниц теста, например, логотип.

Плагин испльзует для отображения модулей теста и просмотра результатов собственные шаблоны, не зависящие от основной темы оформления сайта. Заставка страницы теста позволяет разместить необходимые элементы фирменного стиля на этих страницах.

Просмотр результатов тестирования

Раздел MITEST -> Тесты содержит информацию о завершенных прохождениях теста посетителями сайта. По каждому пользователю фиксируется соответствующий прохождению модуль, дата, тестируемый веб-сайт, набранный бал, ссылка для просмотра профиля ответов (аналогичная ссылка отправляется пользователю на email, если он выбирает соответствующую опцию). В таблице также указывается запрос на консультацию и контактная информация (если были оставлены пользователем).
